## Storybook-First Methodology

### 📖 Principle: "Storybook is the source of truth"

```yaml
Component Lifecycle (Storybook-centric):

1. DESIGN (Figma)
   └─ Export component specs → Storybook stories

2. DEVELOP (TypeScript + React)
   └─ Implement component alongside stories
   └─ TDD: Write stories BEFORE component code

3. DOCUMENT (Storybook Docs)
   └─ Auto-generated API docs from TSDoc
   └─ Usage examples (do's/don'ts)
   └─ Accessibility checklist per story

4. REVIEW (Chromatic CI)
   └─ Visual regression testing
   └─ Accessibility audit (axe-core integration)
   └─ Changes approved before merge

5. RELEASE (NPM + Design System)
   └─ Publish storybook (public documentation)
   └─ Changelog per component
   └─ Migration guides if breaking changes

6. MAINTENANCE
   └─ Story updates = component updates
   └─ Deprecation patterns documented in stories
```

### Story Structure (CSF 3.0 Template)

```typescript
import { Meta, StoryObj } from '@storybook/react';
import { Button } from './Button';
import { expect, within, userEvent } from '@storybook/test';

/**
 * Primary action button for forms & CTAs.
 * Supports variant={primary|secondary|danger}, size={sm|md|lg}
 *
 * @accessibility WCAG 2.1 AA
 *   - Focus outline visible 2px
 *   - Keyboard nav: Tab/Enter/Space
 *   - aria-label required if icon-only
 */
const meta = {
  title: 'Components/Button',
  component: Button,
  tags: ['autodocs'],
  parameters: {
    layout: 'centered',
    docs: {
      description: {
        component: 'Accessible button with multiple variants.',
      },
    },
  },
  argTypes: {
    variant: {
      control: 'select',
      options: ['primary', 'secondary', 'danger'],
      description: 'Visual style variant',
    },
    size: {
      control: 'select',
      options: ['sm', 'md', 'lg'],
      description: 'Button size (mobile-first: sm default)',
    },
    disabled: {
      control: 'boolean',
      description: 'Disabled state (cursor: not-allowed)',
    },
  },
} satisfies Meta<typeof Button>;

export default meta;
type Story = StoryObj<typeof meta>;

/**
 * Primary button (CTA)
 *
 * ✅ Do: Use for main actions (Submit, Save, Continue)
 * ❌ Don't: Use 3+ primary buttons per section
 */
export const Primary: Story = {
  args: {
    variant: 'primary',
    size: 'md',
    children: 'Click me',
  },
  play: async ({ canvasElement }) => {
    // Visual regression baseline
    await expect(canvasElement).toBeInTheDocument();
  },
};

/**
 * Secondary button (lower priority)
 */
export const Secondary: Story = {
  args: {
    variant: 'secondary',
    children: 'Cancel',
  },
};

/**
 * Danger button (destructive actions)
 *
 * ⚠️ Safety: Requires confirmation dialog before execution
 */
export const Danger: Story = {
  args: {
    variant: 'danger',
    children: 'Delete',
  },
};

/**
 * Keyboard interaction test
 */
export const KeyboardNav: Story = {
  args: {
    variant: 'primary',
    children: 'Keyboard focus me',
  },
  play: async ({ canvasElement }) => {
    const button = within(canvasElement).getByRole('button');

    // Tab focus
    await userEvent.tab();
    await expect(button).toHaveFocus();

    // Enter to activate
    await userEvent.keyboard('{Enter}');
    await expect(button).toHaveAttribute('aria-pressed', 'true');
  },
};

/**
 * Icon-only button (requires aria-label)
 *
 * Accessibility: Icon buttons MUST have screen reader label
 */
export const IconOnly: Story = {
  args: {
    variant: 'primary',
    'aria-label': 'Close dialog',
    children: '✕',
  },
  parameters: {
    docs: {
      description: {
        story: 'Icon button requires explicit aria-label for accessibility.',
      },
    },
  },
};
```

## WCAG 2.1 Accessibility Standards

### Level AA (Minimum Target) → AAA (Excellence)

```yaml
WCAG Principles (POUR):

  P - Perceivable:
    [ ] 1.4.3 Contrast minimum 7:1 (AAA) vs 4.5:1 (AA)
    [ ] Color not only differentiator (patterns / icons too)
    [ ] Alt text for images (descriptive, not "image.png")
    [ ] Keyboard access indicators visible

  O - Operable:
    [ ] 2.1 Keyboard navigation (Tab / Shift+Tab / Enter / Esc / Arrow keys)
    [ ] 2.4 Focus order logical (source order ≈ visual order)
    [ ] 2.5 Touch targets 44x44px minimum
    [ ] Trap prevention (keyboard shouldn't get stuck in modal)

  U - Understandable:
    [ ] 3.3 Error messages plain language (no codes: "Email invalid" not "ERR_EMAIL_001")
    [ ] 3.3.4 Help & guidance (labels clear, placeholders ≠ labels)
    [ ] Consistent navigation (header position, button styles)

  R - Robust:
    [ ] 4.1 ARIA usage correct (no aria-label on <button> with text)
    [ ] Semantic HTML (use <button> not <div onclick>)
    [ ] Mobile & AT compatibility (tested with screen readers)
```

### Automated a11y Testing (axe DevTools + Storybook)

```typescript
// Storybook integration: stories/Button.stories.tsx
import { injectAxe, checkA11y } from 'axe-playwright';

export const Accessible: Story = {
  args: { variant: 'primary', children: 'Click' },
  play: async ({ canvasElement }) => {
    await injectAxe(canvasElement);
    await checkA11y(canvasElement, null, {
      rules: {
        'color-contrast': { enabled: true }, // AAA level
        'button-name': { enabled: true }, // Icon buttons need aria-label
      },
    });
  },
};
```

### Manual a11y Checklist (Per Component)

```yaml
Component a11y Audit Template:

□ VISUAL:
  - High contrast mode ✓ (Windows High Contrast)
  - Font scaling 200% ✓ (text readable)
  - Color blind safe ✓ (no red/green only diff)

□ KEYBOARD:
  - Tab order logical ✓ (follow visual flow)
  - All interactive elements reachable ✓
  - Focus outline visible 2px+ ✓
  - Escape key closes modals ✓
  - Arrow keys for select/menu ✓

□ ARIA:
  - Semantic HTML (no div buttons) ✓
  - aria-label if icon-only ✓
  - aria-describedby for error msgs ✓
  - Role attributes correct ✓
  - No aria-* misuse ✓

□ SCREEN READER:
  - NVDA (free, Windows) ✓
  - JAWS (premium, Windows) ✓
  - VoiceOver (Mac/iOS built-in) ✓
  - TalkBack (Android built-in) ✓
  - Purpose clear without visual context ✓

□ MOBILE:
  - Touch targets 44x44px ✓
  - No hover-only info ✓
  - Zoom doesn't break layout ✓
```

### TypeScript Strict Generics

```typescript
// ✅ Polymorphic component (accepts data types)
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
  keyExtractor: (item: T) => string;
  onSelect?: (item: T) => void;
}

export function List<T>({
  items,
  renderItem,
  keyExtractor,
  onSelect,
}: ListProps<T>) {
  return (
    <ul role="listbox">
      {items.map((item) => (
        <li
          key={keyExtractor(item)}
          role="option"
          onClick={() => onSelect?.(item)}
        >
          {renderItem(item)}
        </li>
      ))}
    </ul>
  );
}

// Usage: Type-safe, no casting needed
<List
  items={users}
  renderItem={(user) => `${user.name} (${user.email})`}
  keyExtractor={(user) => user.id}
  onSelect={(user) => console.log(user.email)}
/>
```
